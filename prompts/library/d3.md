# D3.js

D3 (Data-Driven Documents) is the low-level standard for bespoke, SVG-based data
visualization on the web — bind data to the DOM and apply data-driven
transformations. This is one of anyplot's first JavaScript-language entries; the
file extension is **`.js`** and the runtime is the **browser render harness**
(`node automation/js-render/render.mjs`), **not** Python, Rscript, or julia.

D3 charts render in a browser DOM — there is no headless CLI that writes a PNG.
You author an **idiomatic D3 snippet** that builds an `<svg>` inside the mount
node `#container`; the harness wraps it in an HTML page, loads the pinned `d3`
bundle, renders it under headless Chromium (Playwright), and screenshots the
mount node at the exact canvas size. It emits **both** `plot-{theme}.png` and
`plot-{theme}.html`. You never write to the filesystem.

> **Why a real browser, not jsdom.** jsdom does not implement SVG text
> measurement (`getBBox` / `getComputedTextLength` return 0), so axis ticks and
> labels would overlap. The harness uses a real layout engine (Chromium) so D3's
> auto-layout is faithful.

## IMPORTANT: No Workarounds

**If a spec cannot be implemented with D3 + the in-DOM SVG it builds, DO NOT
shell out to Chart.js, ECharts, or any other library.** D3 is maximally
flexible; almost any 2-D static chart is expressible. If the spec's primary
value is genuine interactivity (hover/zoom/brush/drag) with no meaningful static
form, return `NOT_FEASIBLE`. Do not simulate interaction (fake tooltips,
pretend hover states).

D3 submodules available in the pinned bundle: the full default `d3` global
(`d3-selection`, `d3-scale`, `d3-axis`, `d3-shape`, `d3-array`, `d3-hierarchy`,
`d3-format`, `d3-time`, `d3-interpolate`, etc.). Do **not** `import` extra
packages — `d3` is a single global.

## The mount-node contract (how your snippet connects to the harness)

The harness gives you a pre-sized `<div id="container">` and these globals (the
browser has no `process.env`, so this is the JS analogue of `ANYPLOT_THEME`):

```js
window.ANYPLOT_THEME      // "light" | "dark"
window.ANYPLOT_TOKENS     // { pageBg, elevatedBg, ink, inkSoft, grid, palette[8], amber, seq[2], div[3] }
window.ANYPLOT_SIZE       // { width, height } — the CSS mount size you size the <svg> to
window.d3                 // the d3 global (already loaded)
```

Your snippet must:
1. Read `const { width, height } = window.ANYPLOT_SIZE`.
2. `d3.select("#container").append("svg").attr("width", width).attr("height", height)`.
3. Draw with margins, scales, axes, and shapes inside that svg.

Size the `<svg>` to `window.ANYPLOT_SIZE` — never hard-code other pixel
dimensions. The harness screenshots `#container` at `deviceScaleFactor: 2`, so
an SVG sized to the CSS mount lands on exact target pixels automatically.

## Canvas — hard rule, no deviation

The saved PNG must be **exactly** one of these two sizes (the `impl-review.yml`
gate rejects anything off by more than 16 px and re-triggers repair):

| Orientation | Pixels      | CSS mount (`window.ANYPLOT_SIZE`) | How                          |
|-------------|-------------|-----------------------------------|------------------------------|
| Landscape   | 3200 × 1800 | 1600 × 900                        | default                      |
| Square      | 2400 × 2400 | 1200 × 1200                       | `//# anyplot-orientation: square` |

For a square spec, put this on its own line near the top of the file:

```js
//# anyplot-orientation: square
```

Then build the svg from `window.ANYPLOT_SIZE` so it matches the chosen
orientation.

## Reproducibility

Generate data in-memory and deterministically. There is no seeded RNG in the
browser; use a tiny fixed-seed LCG or hard-code the data. **Never `fetch()`,
`d3.csv`, `d3.json`, or load from a CDN/network** — the runtime is offline and
the HTML must be self-contained. Build arrays in code.

## Colors — Imprint palette (from `window.ANYPLOT_TOKENS`)

The first categorical series is **always** `#009E73` (`tokens.palette[0]`). Data
colors are **identical** between light and dark — only chrome flips.

```js
const t = window.ANYPLOT_TOKENS;
// categorical scale
const color = d3.scaleOrdinal().domain(categories).range(t.palette);
// continuous sequential — interpolate between the two Imprint stops
const seq = d3.scaleSequential(d3.interpolateRgbBasis(t.seq)).domain([min, max]);
// diverging — three Imprint stops (midpoint already theme-adaptive)
const div = d3.scaleSequential(d3.interpolateRgbBasis(t.div)).domain([-m, m]);
```

Never use `d3.interpolateViridis`/`Turbo`/`Rainbow` or any built-in D3 color
scheme — only the Imprint stops above.

## Theme-adaptive chrome (D3 mapping)

Map tokens onto fills/strokes. The `#container` background is already
`t.pageBg`; do not paint a white rect over it.

```js
const t = window.ANYPLOT_TOKENS;
// axis text + ticks
svg.selectAll(".tick text").attr("fill", t.inkSoft).style("font-size", "14px");
svg.selectAll(".tick line").attr("stroke", t.grid);
svg.selectAll(".domain").attr("stroke", t.inkSoft);
// title
svg.append("text").attr("fill", t.ink).style("font-size", "22px") /* … */;
// gridlines: stroke t.grid
```

## Script Skeleton

```js
//# anyplot-orientation: landscape
const t = window.ANYPLOT_TOKENS;
const { width, height } = window.ANYPLOT_SIZE;
const margin = { top: 80, right: 60, bottom: 70, left: 80 };
const iw = width - margin.left - margin.right;
const ih = height - margin.top - margin.bottom;

// --- Data (in-memory, deterministic) ---------------------------------------
const data = [
  { label: "Mon", value: 12 }, { label: "Tue", value: 19 }, { label: "Wed", value: 7 },
  { label: "Thu", value: 15 }, { label: "Fri", value: 11 },
];

// --- SVG mount --------------------------------------------------------------
const svg = d3.select("#container").append("svg").attr("width", width).attr("height", height);
const g = svg.append("g").attr("transform", `translate(${margin.left},${margin.top})`);

// --- Scales -----------------------------------------------------------------
const x = d3.scaleBand().domain(data.map((d) => d.label)).range([0, iw]).padding(0.2);
const y = d3.scaleLinear().domain([0, d3.max(data, (d) => d.value)]).nice().range([ih, 0]);

// --- Axes -------------------------------------------------------------------
const xAxis = g.append("g").attr("transform", `translate(0,${ih})`).call(d3.axisBottom(x));
const yAxis = g.append("g").call(d3.axisLeft(y));
for (const ax of [xAxis, yAxis]) {
  ax.selectAll("text").attr("fill", t.inkSoft).style("font-size", "14px");
  ax.selectAll("line").attr("stroke", t.grid);
  ax.select(".domain").attr("stroke", t.inkSoft);
}

// --- Bars -------------------------------------------------------------------
g.selectAll("rect").data(data).join("rect")
  .attr("x", (d) => x(d.label)).attr("y", (d) => y(d.value))
  .attr("width", x.bandwidth()).attr("height", (d) => ih - y(d.value))
  .attr("fill", t.palette[0]);

// --- Title ------------------------------------------------------------------
svg.append("text").attr("x", width / 2).attr("y", 44).attr("text-anchor", "middle")
  .attr("fill", t.ink).style("font-size", "22px").style("font-weight", "600")
  .text("bar-basic · javascript · d3 · anyplot.ai");
```

## D3-Specific Gotchas

- Size the `<svg>` from `window.ANYPLOT_SIZE`; never hard-code 960×500 or other
  tutorial defaults — they fail the canvas gate.
- D3 renders synchronously; the harness settles fonts + a double rAF, so no
  manual delay is needed. Do not add transitions/animations to the static draw.
- No `import`/`require`, no `<script src=…>`, no `d3.csv`/`fetch` — `d3` is a
  global and data is built in code.
- Use `font-size` in **px** (the canvas is large); 14–22 px reads well at
  3200 px wide.
- Append exactly one `<svg>` to `#container`.
